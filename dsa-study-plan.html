DSA roadmap for a React / React Native developer

8 weeks, 240 attempts, 150 core LeetCode problems

This plan is built from the NeetCode 150 pattern order and tuned for someone with real shipping experience. The unique set gives you the core patterns; the review slots push the total attempt count into your 200-300 goal. The links point to the canonical LeetCode problems so you can move fast and stay organized.

150 unique LeetCode problems in the core set
240 total attempts target across 8 weeks
90 review / re-solve attempts to lock in patterns
8 pattern-heavy weeks with a final mixed revision block

How to use the plan

Each day, do one focused block of new problems and one smaller block of review. If a problem takes too long, read the hint or solution, then close the page and solve it again from scratch. That second solve is where the pattern starts to stick.

1

Recognize the pattern first

Before coding, ask whether the problem is really a window, pointer, stack, tree, graph, or DP question.

2

Limit the first attempt

Give yourself 20 to 30 minutes. If you are stuck, get unstuck quickly, then re-derive the answer yourself.

3

Re-solve the misses

At the end of the day, re-open one missed question and solve it without looking. This is how speed builds.

4

Track the takeaway

Write one short note: pattern, invariant, and the key trick. A tiny notebook beats vague memory every time.

The weekly attempt budget is fixed at 30 attempts, which is aggressive but realistic for your 2-month target.
